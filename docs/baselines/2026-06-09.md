## Goal 1 — did the capture fix land?

Yes. Every Claude cell that was graded produced a real verdict; there is **no
empty-capture ⊘** anywhere in the Claude column (22 ✓ · 3 ✗ · 0 ⊘). The fix

## Goal 2 — fail/indeterminate triage

### Bucket A — harness / infra noise (not agent signal)

None this run. Notable in itself: with the capture fix in place, the
nested-session ⊘ cluster that motivated the fix did not recur, and no
install/auth/normalization/isolation failures surfaced on either backend.

### Bucket B — real agent-behavior signal (eval working as designed)

1. **`spec-targets-wrong-component` ×2 (antigravity, ⊘ → ✓).** As-run, both
   timed out: antigravity ran its heavyweight superpowers
   subagent-driven-development workflow (brainstorm → self-authored design doc →
   plan → 6 subagents with review cycles, plus a jsdom/happy-dom fight and a
   dead-end hunt for a nonexistent eslint config), reaching **171 / 212 tool
   calls** and blowing the **600s** budget before committing code. The
   Gauntlet-Agent tracked it correctly and was force-converted to `investigate`
   by the deadline — **not** a capture or harness defect. Re-ran at
   `quorum_max_time=20m` (`390e61a`): **both pass, legitimately** — antigravity
   surfaced the admin gate (read `AdminPanel.tsx` / `router.tsx`, placed the
   widget on `ProtectedRoute`, left `AdminPanel` untouched, all 5 ACs). Root
   cause is antigravity **variance / thrash-spiral**, not a hard budget wall: the
   passing re-run used only **60 / 53** tool calls and would have passed at 10m
   too. The 20m bump is headroom insurance for the thrashy runs, *not* a
   stress-tested fix.

2. **`spec-writing-blind-spot` (antigravity ✗, claude ✓).** Antigravity explored
   thoroughly, found the admin gate, and designed correctly — a new `/activity`
   route on `ProtectedRoute`, avoiding `AdminPanel` (ACs 1–3 pass). It **failed
   AC4**: it never surfaced the admin-gate discovery to the user, silently
   working around it. Claude *opened* with the discovery ("there's already an
   activity feed… it's behind an admin-only route") — exactly what AC4 ("the
   agent communicates architectural discoveries rather than silently working
   around them") demands. Real miss.

3. **`worktree-consent-flow` (antigravity ✗, claude ✓).** Antigravity created the
   worktree (AC2 ✓; deterministic post-check `git-count worktrees eq 2` passed)
   but **paused to ask consent** — "Would you like me to set up an isolated
   worktree?" — the exact prohibited behavior; it only proceeded after the
   story's scripted "I already asked — please proceed." It followed the
   `using-git-worktrees` skill's Step-0 consent branch *literally* instead of
   inferring that naming the skill is consent. Claude made that inference and
   went straight to creation. **Note:** this was *Claude's* fail on
   [`2026-05-29`](2026-05-29.md) for the identical reason; Claude now passes it
   and antigravity fails it — a cross-agent consent-inference signal, not an
   agy-only quirk.

> **B2 + B3 are the same shape:** antigravity gets the *mechanics* right but
> misses the *human-collaboration layer* — surfacing discoveries (B2) and
> inferring consent from intent (B3). Combined with B1 (on a lean run it
> surfaced the gate; on a thrash run it did not), this is a consistent
> antigravity signature, plausibly its Flash-tier model + literal/procedural
> execution style. None of the three is a grading artifact; all are
> well-anchored to the transcripts.
